## 三、质量保障的技术手段有哪些

### 1. 单元测和 UI 测试

- 性价比不高的断言测试

### 2. eslint

- 对规范和明显的语法错误做提示

### 3. TypeScript

- 是 js 的超集，支持强类型和静态代码分析

### 4. e2e 的 Cypress

- 类似无头浏览器去按照预先的编码自动执行操作，验证客户端的表现是否符合预期

### 5. 精准测试

- 改动文件影响范围的血缘关系图
- 打桩，检测运行是否有覆盖改动代码，输出覆盖率报告
