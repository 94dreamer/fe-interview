## 三、质量保障的技术手段有哪些

### 1. 单元测和 UI 测试

- 性价比不高的断言测试

### 2. eslint

- 对规范和明显的语法错误做提示

### 3. TypeScript

- 是 js 的超集，支持强类型和静态代码分析

### 4. e2e 的 Cypress

- 类似无头浏览器去按照预先的编码自动执行操作，验证客户端的表现是否符合预期

### 5. 精准测试

- 改动文件影响范围的血缘关系图
- 打桩，检测运行是否有覆盖改动代码，输出覆盖率报告

### 6. 错误监控

- 通过收集客户端的运行错误并上报来提前感知问题和还原情况

## 四、CSS 在性能优化方面的实践

- css 压缩与合并 Gizp 压缩
- css 文件放到 head 里，不要用@import
- 初始化样式尽量提升（可以考虑少部分关键 css 使用内部样式表）
- 选择器使用合理 适度嵌套
- 尽量使用缩写 减少代码量
